# Benchmark Methodology v1

Version: 1.0
Date: 2026-03-29
Status: Active — all runs must reference this version.

## Purpose

This document defines the rules for running benchmark evaluations against
the rehydration kernel. Any run that produces evidence for claims must
follow this methodology exactly. Deviations must be documented as a new
methodology version.

## Single-Variable Rule

During a comparison, **only one variable changes at a time**:

- Parser configuration OR temperature OR max_tokens OR prompt OR dataset
  OR planner version — never two simultaneously.

If multiple variables changed between runs, the comparison is observational,
not experimental. Label it accordingly.

**Sampling (Qwen3 model card):**
- Thinking mode: `temperature=0.6`, `top_p=0.95`
- Non-thinking mode: `temperature=0.7`, `top_p=0.8`
- DO NOT use greedy decoding (`temperature=0.0`) — causes repetition loops

## Calibration (mandatory)

Every run must pass calibration before any evaluation:

1. **Judge calibration** (`calibrate_judge`): sends known-good and known-bad
   synthetic responses. Judge must correctly score both. If either fails,
   run aborts with zero API cost.

2. **Agent calibration** (`calibrate_agent`): sends a minimal prompt and
   checks for non-empty JSON response with required fields (`failure_point`,
   `restart_node`, `reason`, `reason_source`, `confidence`). If empty or
   malformed, run aborts.

## Dataset

Generated by `dataset_generator.rs`. Parameters:

| Parameter | Values |
|-----------|--------|
| Scales | `micro` (4 nodes), `meso` (21 nodes), `stress` (49 nodes) |
| Domains | `Operations`, `SoftwareDebugging` |
| Relation mixes | `explanatory`, `structural`, `mixed` |
| Noise modes | `clean` (structural), `competing-causal`, `conflicting-path`, `competing-restart` |
| Seeds per cell | 3 (rotated node-kind order + varied rationale text) |

Noise modes are distributed across mixes (not fully crossed):
- `explanatory` → clean + competing-causal
- `structural` → clean + conflicting-path
- `mixed` → clean + competing-restart

Total variants per run: 3 scales × 2 domains × 3 mixes × 2 noise × 3 seeds = **108**.

## Metrics

**Judge-evaluated (per eval):**

| Metric | Definition |
|--------|-----------|
| `task` | Did the agent identify the correct failure point? |
| `restart` | Did the agent identify the correct recovery node? |
| `restart_exact` | Exact node match |
| `restart_off_by_one` | ±1 hop from correct |
| `restart_explained` | Recovery justified with causal rationale |
| `reason` | Was the rationale preserved from context? |
| `reason_correct` | Rationale matches main causal chain |
| `reason_distractor` | Rationale came from noise branch |

**Evaluator-computed (deterministic, no judge):**

| Metric | Definition |
|--------|-----------|
| `llm_reason_source` | Agent's declared source: `graph_metadata` / `inferred` / `not_available` |
| `llm_confidence` | Agent's declared confidence: `high` / `medium` / `low` |
| `llm_reason_fabricated` | `true` when `reason_source == "graph_metadata" AND causal_density == 0.0` |

**Kernel-computed (per render):**

| Metric | Definition |
|--------|-----------|
| `rendered_tokens` | cl100k_base token count of rendered context |
| `compression_ratio` | raw_equivalent_tokens / rendered_tokens |
| `causal_density` | explanatory relations / total relations |
| `noise_ratio` | noise nodes / total nodes |
| `resolved_mode` | Planner decision: `reason_preserving` or `resume_focused` |

## Execution

### Pre-run checklist

1. vLLM server running with correct model + `--reasoning-parser=qwen3`
2. API keys set: `ANTHROPIC_KEY`, `OPENAI_KEY`
3. TLS certs present: `/tmp/vllm-client.crt`, `/tmp/vllm-client.key`
4. Docker/Podman available for testcontainers
5. Methodology version noted in run manifest

### Run command

```bash
EVAL_MATRIX_PATH=<config.yaml> \
cargo test -p rehydration-tests-paper --features container-tests \
  --test llm_judge_prompt_evaluation -- --nocapture --test-threads=1
```

### Filters (subset runs)

```bash
FILTER_MODELS="qwen3-8b" FILTER_JUDGES="sonnet-4.6" FILTER_PROMPTS="default"
FILTER_SCALES="micro" FILTER_NOISE="clean"
```

### Token budget override

```bash
BENCHMARK_TOKEN_BUDGET=512  # default: 4096
```

## Artifacts

Every run produces a timestamped directory:

```
artifacts/e2e-runs/YYYY-MM-DD_HHMMSS/
├── test.log          full stderr
├── summary.json      all results as JSON array
├── report.md         markdown table
└── results/          one JSON per eval
    ├── 0001_agent_judge_prompt_variant.json
    └── ...
```

Each result JSON contains all metrics listed above plus `agent_response`
and `judge_raw` for post-hoc analysis.

## Reproducibility

A reproducible run requires:
- Matrix YAML (e.g. `paper-recalc-gpt54.yaml`)
- Kernel commit SHA
- vLLM image tag + server args
- API model IDs (without date suffixes if pinned)

Store the matrix YAML in the repo alongside the artifacts.

## Statistical Analysis

- **Confidence intervals**: Wilson score for proportions, bootstrap for gaps
- **Null hypothesis**: H0 = no difference between explanatory and structural
- **Rejection criterion**: lower bound of 95% gap CI > 0
- **Seeds**: minimum 3 per cell for variance estimation
- **Cross-judge**: same data evaluated by 2+ judges for robustness

## Exclusion Criteria

Exclude an eval from analysis if:
- Agent returned empty response (calibration should catch this)
- Judge returned unparseable JSON
- Infrastructure error (container crash, network timeout)

Document exclusions in the run report.
